개요

Overview

-3W Booking Dashboard는 출항 3주 전(WOS-3) 부킹 현황을 모니터링하는 대시보드입니다.

  • ① 소석률 현황 — BSA 대비 실선적(Normal) 비율, 3주전BKG/BSA 비중
  • ② 부킹 트렌드 — 지역별/화주별/영업사원별 실적 + 고수익 리드타임 분석
  • ③ 전환 분석 — 캔슬률 분석, 화주별 캔슬 Top 15
  • ④ 수익성 — CM1/TEU Top/Bottom, 수익 집중도
  • ⑤ 부킹 패턴 — WOS 단계별 부킹 접수 시점 분석

기본 필터: 현재 주 +3주 후의 월 및 해당 주차로 자동 설정됩니다.

-3W Booking Dashboard monitors booking status 3 weeks before sailing (WOS-3).

  • ① Occupancy — LFT(Normal)/BSA ratio, WOS-3 BKG/BSA ratio
  • ② Trends — By region/shipper/salesperson + high-profit lead time analysis
  • ③ Conversion — Cancel rate analysis, Top 15 cancel shippers
  • ④ Profitability — CM1/TEU Top/Bottom, revenue concentration
  • ⑤ Patterns — WOS stage booking timing analysis

Default filter: automatically set to the month & week of current week + 3 weeks ahead.

과거 데이터 조회

Historical Data

헤더 우측의 날짜 드롭다운을 사용하여 과거 시점의 대시보드 데이터를 조회할 수 있습니다.

사용법

  1. 로그인 후 우측 상단에 날짜 선택 드롭다운이 표시됩니다
  2. 기본값은 최신 데이터 (오늘 생성된 스냅샷)
  3. 드롭다운을 열면 Google Drive에 저장된 과거 날짜 목록이 자동으로 로드됩니다
  4. 원하는 날짜를 선택하면 해당 시점의 데이터로 대시보드가 갱신됩니다

동작 방식

선택데이터 소스설명
최신 (기본)GitHub Pages data.json매일 09:03 자동 업데이트된 최신 스냅샷
과거 날짜Google Drive해당 날짜에 생성된 dashboard_summary_YYYYMMDD.json 파일
활용 예시: 지난주와 이번주의 소석률 변화를 비교하거나, 특정 시점의 부킹 현황을 확인할 때 유용합니다. 최대 30개의 과거 날짜가 표시됩니다.

Use the date dropdown in the top-right header to view dashboard data from past dates.

How to Use

  1. After login, a date selector dropdown appears in the top-right corner
  2. Default is Latest data (today's snapshot)
  3. Open the dropdown to see past dates automatically loaded from Google Drive
  4. Select a date to reload the dashboard with that day's data

How It Works

SelectionData SourceDescription
Latest (default)GitHub Pages data.jsonLatest snapshot, auto-updated daily at 09:03
Past dateGoogle Drivedashboard_summary_YYYYMMDD.json from that date
Use case: Compare this week's occupancy vs last week, or review booking status at a specific point in time. Up to 30 past dates are shown.

WOS (Week of Sailing) 상세

WOS Details

WOS = Week of Sailing. Booking Schedule의 주 시작일(일요일) - 부킹 접수일로 계산합니다.

단계기준의미예시
WOS-3≥ 15일3주+ 전 부킹4/1 부킹 → 4/26 출항 (25일)
WOS-28~14일2주 전 부킹4/15 부킹 → 4/26 출항 (11일)
WOS-11~7일1주 전 부킹4/20 부킹 → 4/26 출항 (6일)
WOS< 1일출항 주 부킹4/26 부킹 → 4/26 출항 (당일)
WOS-3 비중이 높을수록 좋습니다. 조기 부킹 = 선복 계획 안정 + 캔슬 위험 감소 + 높은 수익성

WOS = Week of Sailing. Calculated as BKG Schedule week start (Sunday) minus Booking date.

StageLead TimeMeaningExample
WOS-3≥ 15 days3+ weeks aheadBooked Apr 1 → Sails Apr 26
WOS-28~14 days2 weeks aheadBooked Apr 15 → Sails Apr 26
WOS-11~7 days1 week aheadBooked Apr 20 → Sails Apr 26
WOS< 1 daySailing weekBooked Apr 26 → Sails Apr 26
Higher WOS-3 = better. Early bookings → stable capacity + lower cancel risk + higher profitability

고수익화주 / 구간별 고수익 부킹분류

Shipper & Route Profit Classification

선적지(POR Port)별 화주 CM1/TEU를 평균과 비교하여 분류합니다.

태그기준
고수익화주화주 CM1/TEU ≥ 선적지 평균
저수익화주화주 CM1/TEU < 선적지 평균

태그 산정 우선순위

  1. 전월 + 동일 선적지 CM1/TEU vs 평균
  2. 전월 + 전체 선적지 CM1/TEU vs 전체 평균
  3. 당월 + 동일 선적지
  4. Grade C+D → 고수익, A+B → 저수익

동일 선적지+화주 조합에 하나의 태그만 부여됩니다 (최신월 기준).

구간별 고수익 부킹분류

POR Port + DLY Port 루트별 평균 CM1/TEU 대비 화주별 비교로 개별 부킹을 분류합니다.

분류기준
구간별 고수익화주 CM1/TEU ≥ 루트(POR+DLY) 평균
구간별 저수익화주 CM1/TEU < 루트 평균
고수익화주는 선적지 기준 화주 단위 태그, 구간별 고수익은 루트(선적지+도착지) 기준 부킹 단위 분류입니다.

Compares shipper CM1/TEU against POR port average.

TagCriteria
Hi-ShipperShipper CM1/TEU ≥ Port average
Lo-ShipperShipper CM1/TEU < Port average

Tag Priority

  1. Prev month + same POR port vs average
  2. Prev month + all ports vs overall average
  3. Current month + same POR
  4. Grade C+D → High, A+B → Low

One tag per POR+shipper combination (latest month basis).

Route-based Booking Classification

Compares shipper CM1/TEU against POR+DLY route average to classify individual bookings.

ClassCriteria
Route HighShipper CM1/TEU >= Route (POR+DLY) average
Route LowShipper CM1/TEU < Route average
Hi-Shipper = POR-level shipper tag. Route High = POR+DLY route-level booking classification.

자동 업데이트

Auto Updates

매일 09:03 자동 실행

  1. Tableau에서 1.csv + 2.csv + BSA 다운로드
  2. Booking Snapshot 처리 (수식, 필터, 고수익태그)
  3. JSON 생성 → Google Drive 업로드 + dist/data.json 복사
  4. GitHub Pages 자동 push → 웹 배포 완료

데이터 범위

파일범위
1.csv (Booking_schedule)2025-12-28 ~ 현재 주 +4주
2.csv (Date_vsl)전체
BSAYYYY 2025~2027, YYYYMM 전체

Daily at 09:03 (auto)

  1. Download from Tableau (1.csv + 2.csv + BSA)
  2. Process booking snapshot (formulas, filters, profit tags)
  3. Generate JSON → Google Drive + dist/data.json
  4. Auto push to GitHub Pages

Data Range

1.csv: 2025-12-28 ~ current week +4 weeks | 2.csv: Full | BSA: 2025-2027 all months
