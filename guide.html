개요

Overview

-3W Booking Dashboard는 출항 3주 전(WOS-3) 부킹 현황을 모니터링하는 대시보드입니다.

  • ① 소석률 현황 — BSA 대비 실선적(Normal) 비율, 3주전BKG/BSA 비중
  • ② 부킹 트렌드 — 지역별/화주별/영업사원별 실적 + AB vs CD 등급 분석
  • ③ 전환 분석 — 캔슬률 분석, 화주별 캔슬 Top 15
  • ④ 수익성 — CM1/TEU Top/Bottom, 수익 집중도
  • ⑤ 부킹 패턴 — WOS 단계별 부킹 접수 시점 분석

기본 필터: 현재 주 +3주 후의 월 및 해당 주차로 자동 설정됩니다.

3주전 부킹이 실제 선적으로 잘 전환되는가?③ 전환 분석실선적률, 캔슬률, 캔슬 화주

권장 사용 순서

  1. 필터에서 팀/월/주차/도착국가를 맞춥니다.
  2. ① 소석률에서 전체 물량과 선복 대비 부족/초과 여부를 봅니다.
  3. ② 부킹 트렌드에서 부족 또는 초과를 만든 지역, 화주, 영업사원을 찾습니다.
  4. ③ 전환 분석에서 캔슬과 실선적 전환 리스크를 확인합니다.
  5. ④ 수익성과 ⑤ 부킹 패턴으로 물량의 질과 조기 확보 여부를 점검합니다.

-3W Booking Dashboard monitors booking status 3 weeks before sailing (WOS-3).

  • ① Occupancy — LFT(Normal)/BSA ratio, WOS-3 BKG/BSA ratio
  • ② Trends — By region/shipper/salesperson + AB vs CD grade analysis
  • ③ Conversion — Cancel rate analysis, Top 15 cancel shippers
  • ④ Profitability — CM1/TEU Top/Bottom, revenue concentration
  • ⑤ Patterns — WOS stage booking timing analysis

Default filter: automatically set to the month & week of current week + 3 weeks ahead.

Questions This Dashboard Answers

QuestionStart HereWhat to Check
Is volume sufficient against capacity?① OccupancyNormal LFT/BSA, WOS-3 BKG/BSA, monthly trend
Which region, shipper, or salesperson is driving volume?② TrendsBKG and LFT rate by destination/origin, shipper, and salesperson
Does WOS-3 BKG convert to actual shipment?③ ConversionLFT rate, cancel rate, cancel-heavy shippers
Is the volume profitable?④ ProfitabilityCM1/TEU, top/bottom shippers, revenue concentration
Are bookings coming early enough?⑤ PatternsWOS-3/2/1/WOS mix and early-booking share

Recommended Workflow

  1. Set team/month/week/destination filters.
  2. Use ① Occupancy to identify capacity shortage or excess.
  3. Use ② Trends to find the region, shipper, or salesperson behind the movement.
  4. Use ③ Conversion to check cancel and shipment-conversion risk.
  5. Use ④ Profitability and ⑤ Patterns to judge quality and early-booking stability.

필터 해석 기준

  • 팀/선적지/선적포트/도착지/도착포트는 물리적인 운송 루트 범위를 좁히는 필터입니다. 지역별, 소석률, 수익성 화면에 모두 큰 영향을 줍니다.
  • 보기는 지역을 묶는 축을 바꿉니다. 도착지별은 시장/수요 관점, 선적지별은 영업/공급 출발지 관점으로 읽으면 됩니다.
  • 주차를 선택하면 선택 월은 해당 주 실적, 과거 월 비교는 주평균으로 표시됩니다. 특정 주의 상황을 과거 월 평균과 비교할 때 사용합니다.
  • 화주구분/등급/영업사원은 주로 화주 기반 데이터에 강하게 적용됩니다. 영업사원 필터는 `shipper` 데이터 기반 화면에서 가장 정확합니다.
추천: 처음에는 팀+월만 선택해서 전체 구조를 보고, 이상치가 보이면 도착국가/도착포트/영업사원 순서로 좁히는 방식이 가장 빠릅니다.

How to Interpret Filters

  • Team/origin/origin port/destination/destination port narrow the physical route scope and affect occupancy, trend, and profitability views.
  • View changes the regional grouping axis. Destination view is useful for market demand; origin view is useful for sales/supply origin management.
  • Week changes the selected month to actual weekly data while past months are shown as weekly averages. Use it to compare one target week against historical monthly averages.
  • Profit/grade/salesperson are most precise in shipper-based views because those fields live in the shipper dataset.
Recommended: Start with team + month, then narrow by destination country, destination port, and salesperson only after you see an abnormal pattern.

Historical Data

헤더 우측의 날짜 드롭다운을 사용하여 과거 시점의 대시보드 데이터를 조회할 수 있습니다.

사용법

  1. 로그인 후 우측 상단에 날짜 선택 드롭다운이 표시됩니다
  2. 기본값은 최신 데이터 (오늘 생성된 스냅샷)
  3. 드롭다운을 열면 Google Drive에 저장된 과거 날짜 목록이 자동으로 로드됩니다
  4. 원하는 날짜를 선택하면 해당 시점의 데이터로 대시보드가 갱신됩니다

동작 방식

선택데이터 소스설명

활용 예시: 지난주와 이번주의 소석률 변화를 비교하거나, 특정 시점의 부킹 현황을 확인할 때 유용합니다. 최대 30개의 과거 날짜가 표시됩니다.

Use the date dropdown in the top-right header to view dashboard data from past dates.

  2. Default is Latest data (today's snapshot)

  4. Select a date to reload the dashboard with that day's data

월별 BKG vs BSA 차트

보라 바=3주전BKG, 파란 바=실선적(Normal), 회색 바=BSA

빨간 실선=소석률(Normal/BSA), 노란 점선=3주전BKG/BSA, 보라 점선=고수익화주부킹비중(고수익화주3주전BKG/3주전BKG)

실제 계산 결과가 100%를 넘으면 그대로 표시합니다. BKG/BSA는 BSA보다 부킹이 큰 경우, 실선적률은 최종 LST_TEU가 최초 FST_TEU보다 커진 경우 100%를 넘을 수 있습니다.

소석률 바 차트 (토글)

3주전BKG/BSA와 Normal/BSA 두 모드를 버튼으로 전환하여 볼 수 있습니다.

기본: 3주전BKG/BSA | 색상: 초록(≥65%), 노랑(50-65%), 빨강(<50%)

사용 방법

  1. 먼저 KPI 카드에서 전체 소석률과 실선적률을 봅니다.
  2. 월별 차트에서 이번 달이 과거 월 대비 높은지 낮은지 확인합니다.
  3. 소석률 바 차트에서 부족한 도착국가/선적지를 찾습니다.
  4. 부족 지역은 ② 부킹 트렌드에서 화주/영업사원 원인을 확인하고, 초과 지역은 BSA 조정 또는 수익성 점검 대상으로 봅니다.

How to Use

  1. Start with the KPI cards for overall occupancy and LFT rate.
  2. Use the monthly chart to compare the current month with prior months.
  3. Use the occupancy bar to find weak destination/origin areas.
  4. For weak areas, move to ② Trends to find the shipper or salesperson cause. For excess areas, review BSA adjustment and profitability.

Tab 2를 읽는 순서

  1. 지역별에서 어느 도착국가/선적지가 물량 변화를 만들었는지 확인합니다.
  2. 화주별에서 해당 지역의 핵심 화주와 등급, 고수익태그, CM1/TEU를 확인합니다.
  3. 영업사원별에서 물량을 담당하는 영업사원과 도착국가 집중도를 봅니다.
  4. AB vs CD에서 대형화주(A+B) 의존도와 중소화주(C+D) 기반의 안정성을 비교합니다.

서브탭별 상세 해석

서브탭주요 질문해석 기준
지역별어느 지역에서 WOS-3 물량이 크거나 부족한가?BKG는 수요 규모, BSA/소석률은 선복 대비 과부족, 실선적률은 부킹 품질을 의미합니다. 구간별고수익%가 높으면 해당 지역 물량의 수익성이 상대적으로 좋습니다.
화주별어떤 화주가 물량과 수익을 만들고 있는가?BKG 큰 화주는 물량 영향도가 높고, 실선적률 낮은 화주는 캔슬/변경 관리 대상입니다. CM1/TEU가 낮은 대형 화주는 수익성 개선 후보입니다.
영업사원별어떤 영업사원이 어느 도착국가에서 물량을 만들고 있는가?영업사원 총성과와 도착국가 매트릭스를 함께 봅니다. 특정 국가 쏠림이 크면 해당 국가의 시장/화주 리스크가 영업사원 성과에 크게 영향을 줍니다.
AB vs CD물량이 대형화주 중심인가, 중소화주 기반인가?A+B 비중이 높으면 물량 확보는 쉽지만 특정 화주 이탈 리스크가 큽니다. C+D 비중이 높으면 기반은 넓지만 실선적률과 캔슬 관리가 중요합니다.

지역별 화면 활용

  • 3주전 BKG 현황은 선택한 보기 기준(도착지별/선적지별)으로 WOS-3 BKG, BSA, 소석률, 실선적률, 구간별고수익을 보여줍니다.
  • 주차별 3주전 BKG는 해당 월 안에서 주차별 물량 변동을 보여줍니다. 특정 주차만 튀면 일시적 선복/선박 스케줄 영향인지 확인합니다.
  • 고수익화주 vs 저수익화주 리드타임은 수익성 좋은 화주가 조기에 부킹하는지 확인하는 화면입니다.
  • 고수익화주 비중 추이는 WOS-3 물량 중 고수익화주 비중의 월별 변화입니다. 물량이 늘어도 이 비중이 떨어지면 수익성 질이 나빠질 수 있습니다.

화주별 화면 활용

  • 등급은 A+B/C+D 기반의 화주 규모 구분이고, 고수익화주는 선적지 기준 CM1/TEU 비교 태그입니다. 두 지표는 서로 다른 기준입니다.
  • 구간별고수익은 루트 평균 대비 해당 부킹의 수익성 분류입니다. 화주가 고수익화주가 아니더라도 특정 구간에서는 구간별고수익일 수 있습니다.
  • CM1/TEU는 실선적된 Normal 물량 기준 수익성입니다. BKG가 큰데 CM1/TEU가 낮으면 영업/운임 조건 점검이 필요합니다.

영업사원별 도착국가 분석

영업사원별 서브탭은 영업사원 단위의 WOS-3 부킹 성과를 먼저 보여준 뒤, 같은 데이터를 도착국가 기준으로 다시 쪼개서 보여줍니다. 포트 단위가 아니라 국가/그룹(dest) 기준이므로, 영업사원별 지역 포트폴리오와 특정 국가 쏠림을 빠르게 확인하는 용도입니다.

영역확인 포인트
영업사원별 실적화주수, WOS-3 BKG, 실선적, 실선적률, 구간별 고수익 비중, CM1/TEU를 영업사원별로 비교합니다.
Top 영업사원 도착국가 구성상위 영업사원별 WOS-3 BKG를 도착국가별 누적 막대로 표시합니다. 한 사람의 부킹이 특정 국가에 집중되어 있는지, 여러 국가로 분산되어 있는지 확인합니다.
영업사원×도착국가 매트릭스행은 영업사원, 열은 도착국가입니다. 셀 값은 WOS-3 BKG TEU이며, 색이 진할수록 해당 조합의 부킹 규모가 큽니다.
영업사원별 도착국가 상세영업사원+도착국가 조합별 화주수, BKG, 실선적, 캔슬, 구간별 고수익, CM1/TEU를 상세 확인합니다.

해석 방법

  • BKG 집중 — 매트릭스에서 특정 셀만 진하면 해당 영업사원의 부킹이 특정 도착국가에 집중된 상태입니다.
  • 실선적률/캔슬률 — 상세 테이블에서 BKG가 큰 조합의 실선적률이 낮거나 캔슬률이 높으면 확정 관리가 필요한 후보입니다.
  • 수익성 — BKG 규모가 커도 CM1/TEU가 낮으면 물량 확대보다 수익성 개선 또는 화주 믹스 점검이 우선일 수 있습니다.
  • 고수익 구성 — 구간별 고수익%가 높은 조합은 해당 영업사원/국가에서 수익성 있는 부킹이 잘 만들어지는 영역입니다.
주의: BSA는 도착국가/포트 전체 선복 기준 데이터이며 영업사원별로 배분되어 있지 않습니다. 따라서 영업사원×도착국가 상세에는 소석률을 넣지 않고, BKG/실선적/캔슬/수익성 지표 중심으로 봅니다.

AB vs CD 분석 가이드

AB vs CD 탭 하단에 데이터 기반 자동 진단이 표시됩니다:

  • A+B 의존도 — A+B(대형화주) 비중 60% 이상 시 집중 리스크 경고
  • 실선적률 격차 — A+B vs C+D 실선적률 차이 10%p 이상이면 C+D 캔슬 관리 필요

③ 전환 분석

③ Conversion Analysis

전환 분석은 3주전에 확보한 부킹이 실제 선적으로 얼마나 남았는지를 보는 화면입니다. 물량이 충분해 보여도 전환률이 낮으면 실제 소석과 매출로 이어지지 않을 수 있습니다.

영역의미사용 방법

해석 포인트

  • BKG는 충분한데 실선적률이 낮음 — 확보한 부킹의 품질이 낮거나 캔슬/변경이 많을 수 있습니다.
  • 캔슬률은 낮은데 실선적률이 낮음 — 캔슬 외 사유(스케줄 변경, 지연, 상태값, LST_TEU 변동)를 확인합니다.
  • 특정 화주 캔슬 집중 — 영업사원과 함께 부킹 확정 프로세스, 반복 패턴, 선복 배정 우선순위를 점검합니다.

④ 수익성 분석

④ Profitability

수익성 분석은 물량의 크기가 아니라 TEU당 얼마의 CM1을 만드는지를 보는 화면입니다. 소석률이 좋아도 CM1/TEU가 낮으면 좋은 물량이라고 보기 어렵습니다.

영역의미사용 방법
지역별 CM1/TEU도착국가/선적지별 TEU당 CM1같은 물량이라도 어느 지역이 더 좋은 수익을 만드는지 비교합니다.
CM1/TEU 월별 추이지역별 수익성의 시간 변화특정 지역의 수익성이 개선/악화되는 방향을 봅니다.
Top 10 / Bottom 10 화주CM1/TEU가 가장 높거나 낮은 화주Top은 유지/확대 후보, Bottom은 운임/조건/화주 믹스 점검 후보입니다.
수익 집중도상위 화주가 전체 CM1에서 차지하는 비중소수 화주 의존도가 큰지 확인합니다. 집중도가 높으면 이탈 리스크가 큽니다.

해석 포인트

  • BKG 높고 CM1/TEU 높음 — 유지/확대 가치가 높은 핵심 물량입니다.
  • BKG 높고 CM1/TEU 낮음 — 선복을 많이 쓰지만 수익성이 낮은 물량입니다. 운임 또는 믹스 점검이 필요합니다.
  • BKG 낮고 CM1/TEU 높음 — 확대 가능성이 있는 고품질 물량입니다.
  • 수익 집중도 높음 — Top 화주 이탈 시 전체 수익 영향이 크므로 대체 화주/지역을 관리합니다.

⑤ 부킹 패턴

⑤ Booking Patterns

부킹 패턴은 물량이 언제 들어오는지를 보는 화면입니다. WOS-3 비중이 높을수록 선복 계획과 영업 예측이 안정적이고, WOS/WOS-1 비중이 높을수록 막판 변동성이 큽니다.

영역의미사용 방법
지역별 WOS 분포도착국가/선적지별 전체 BKG 중 WOS-3/2/1/WOS 비중조기 부킹이 강한 지역과 막판 부킹이 많은 지역을 구분합니다.
100% 스택 바지역별 부킹 접수 시점 비중 시각화지역 간 리드타임 구조를 빠르게 비교합니다.
화주별 WOS 분포화주별 조기/막판 부킹 성향반복적으로 늦게 부킹하는 화주는 선복 예측 리스크 후보입니다.
고수익 vs 저수익 리드타임수익성 그룹별 부킹 시점 비교고수익화주가 조기에 들어오는지, 저수익화주가 막판 물량을 채우는지 확인합니다.

해석 포인트

  • WOS-3 비중 높음 — 조기 확보가 잘 되고 있어 선복 계획 안정성이 높습니다.
  • WOS/WOS-1 비중 높음 — 막판 영업/스팟 물량 의존도가 높고, 예측 안정성이 낮습니다.
  • 고수익화주 WOS-3 비중 하락 — 조기 확보 물량의 수익성이 약해지고 있을 수 있습니다.
  • 특정 화주의 늦은 부킹 반복 — 선복 예약 룰, cut-off 관리, 영업 커뮤니케이션 점검 대상입니다.

WOS (Week of Sailing) 상세

WOS Details

WOS = Week of Sailing. 부킹이 출항 대비 얼마나 앞서 접수되었는지를 나타내는 리드타임 지표입니다.

계산 공식

대시보드의 WOS 분류는 Booking Schedule (예정 출항일) 기준으로 계산됩니다:

  1. week_start (BKG_Sche) = Booking_schedule 날짜가 속한 주의 일요일
    week_start = Booking_schedule - (요일 offset)
    예: Booking_schedule = 4/29(수) → week_start = 4/26(일)
  2. Lead_time = week_start (BKG_Sche) - Booking_date (부킹 접수일)
    diff = week_start(일요일) - 부킹접수일
  3. diff 값에 따라 WOS 단계를 분류:
단계기준 (diff)의미계산 예시
WOS-3≥ 15일3주+ 전 부킹부킹 4/1, 스케줄 4/29 → ws 4/26 → diff=25일
WOS-28~14일2주 전 부킹부킹 4/15, 스케줄 4/29 → ws 4/26 → diff=11일
WOS-11~7일1주 전 부킹부킹 4/20, 스케줄 4/29 → ws 4/26 → diff=6일
WOS< 1일출항 주 부킹부킹 4/27, 스케줄 4/29 → ws 4/26 → diff=-1일

주차 그룹 vs WOS 분류 — 날짜 기준 차이

항목날짜 기준용도
주차 그룹 (행)Actual_Departure_schedule (Date_vsl, 실제 출항일)월별/주차별 테이블의 행 그룹핑
WOS 분류 (열)Booking_schedule (예정 출항일)WOS-3/2/1/WOS 리드타임 단계

즉, 한 부킹의 주차 배정은 실제 출항일(Date_vsl) 기준이고, 리드타임(WOS) 분류는 Booking_schedule 기준입니다.

WOS-3 비중이 높을수록 좋습니다. 조기 부킹 = 선복 계획 안정 + 캔슬 위험 감소 + 높은 수익성
Booking_schedule을 사용하는 이유: Booking_schedule은 부킹 접수 시점에 확정된 예정 출항일로, 부킹 의사결정 시점의 리드타임을 정확히 반영합니다. 실제 출항일(Date_vsl)은 이후 변경될 수 있어 사후적 관점이 됩니다.

WOS = Week of Sailing. Measures how far in advance a booking was placed relative to the sailing date.

Calculation Formula

The dashboard WOS classification is based on Booking Schedule (planned departure date):

  1. week_start (BKG_Sche) = Sunday of the week containing Booking_schedule date
    week_start = Booking_schedule - (weekday offset)
    Example: Booking_schedule = Apr 29 (Wed) → week_start = Apr 26 (Sun)
  2. Lead_time = week_start (BKG_Sche) - Booking_date
    diff = week_start(Sunday) - booking date
  3. Classify WOS stage by diff value:
StageCriteria (diff)MeaningCalculation Example
WOS-3≥ 15 days3+ weeks aheadBooked 4/1, Schedule 4/29 → ws 4/26 → diff=25d
WOS-28~14 days2 weeks aheadBooked 4/15, Schedule 4/29 → ws 4/26 → diff=11d
WOS-11~7 days1 week aheadBooked 4/20, Schedule 4/29 → ws 4/26 → diff=6d
WOS< 1 daySailing weekBooked 4/27, Schedule 4/29 → ws 4/26 → diff=-1d

Week Grouping vs WOS Classification — Different Date Bases

ItemDate BasisPurpose
Week Group (rows)Actual_Departure_schedule (Date_vsl)Row grouping in monthly/weekly tables
WOS Stage (columns)Booking_schedule (planned departure)WOS-3/2/1/WOS lead time classification

In other words, a booking's week assignment is based on actual departure (Date_vsl), while the lead time (WOS) stage is based on Booking_schedule.

Higher WOS-3 = better. Early bookings → stable capacity + lower cancel risk + higher profitability
Why Booking_schedule? Booking_schedule is the planned departure date fixed at booking time, accurately reflecting the lead time at the point of booking decision. Actual departure (Date_vsl) may change later and represents a retrospective view.

Shipper & Route Profit Classification

선적지(POR Port)별 화주 CM1/TEU를 평균과 비교하여 분류합니다.

태그기준
고수익화주화주 CM1/TEU ≥ 선적지 평균
저수익화주화주 CM1/TEU < 선적지 평균

태그 산정 우선순위

  1. 전월 + 동일 선적지 CM1/TEU vs 평균
  2. 전월 + 전체 선적지 CM1/TEU vs 전체 평균
  3. 당월 + 동일 선적지
  4. Grade C+D → 고수익, A+B → 저수익

동일 선적지+화주 조합에 하나의 태그만 부여됩니다 (최신월 기준).

대시보드의 고수익화주 3주전BKG와 고수익화주부킹비중은 이 고수익태그 기준으로 계산합니다.

구간별 고수익 부킹분류

POR Port + DLY Port 루트별 평균 CM1/TEU 대비 화주별 비교로 개별 부킹을 분류합니다.

분류기준
구간별 고수익화주 CM1/TEU ≥ 루트(POR+DLY) 평균
구간별 저수익화주 CM1/TEU < 루트 평균
고수익화주는 선적지 기준 화주 단위 태그, 구간별 고수익은 루트(선적지+도착지) 기준 부킹 단위 분류입니다.

Compares shipper CM1/TEU against POR port average.

TagCriteria

Lo-ShipperShipper CM1/TEU < Port average

Tag Priority

  1. Prev month + same POR port vs average
  2. Prev month + all ports vs overall average
  3. Current month + same POR
  4. Grade C+D → High, A+B → Low

One tag per POR+shipper combination (latest month basis).
